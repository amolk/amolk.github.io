## About

A 3D scene that renders a branching timeline as a graph of flat-shaded nodes connected by curved edges with per-edge particle streams. Visual language is borrowed from the Apple TV *Foundation* Prime Radiant â€” wine background, sage/copper accents, woven-palette particle wisps drifting along edges, sparse bright "glints" driving bloom pinpricks against a dim matte grain.

As of 2026-05-26 the codebase is split into a **public library surface** (`src/lib/` â€” `<Unfold data={...} theme={â€¦} style={â€¦} layout="layered"/>`, plus public types in `src/lib/types.ts` and an `internal/` folder for the scene), and a **demo app** (`src/demo/` â€” `App.tsx`, `demo-data.ts`, `leva-panels.tsx`) which is the original prototype repositioned as the consumer of the library.

The library is **phase-gated**: data + theme + style + layout are wired (phases 2/3/4); interaction callbacks (`onNode*`, `onEdge*`, `onBackgroundClick`), selection/focus state, `onNodeExpand`, `cameraMode`, and the `UnfoldHandle` ref API are accepted on `UnfoldProps` for forward-compatible type-checking but are silently no-op until phases 6â€“10 land them. Public types deliberately import zero `three` â€” positions are plain `[number, number, number]` tuples and colors are hex strings, so the library boundary never leaks a `THREE.Vector3` to consumers.

Stack: React + Three.js via `@react-three/fiber` and `@react-three/drei`, postprocessing for bloom, `leva` for live controls (now a public `theme`/`style` surface, not just dev HUD). Layout is deterministic across reloads (mulberry32-seeded PRNG). Nodes are flat-shaded; particles have their own GLSL vertex+fragment shaders with an EdgeFlow path that mixes up to 8 colors-in-proportions per edge.
