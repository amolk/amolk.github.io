## About

Subproject of playbooks. End-to-end pipeline that fine-tunes a small open-weight model on real Playbooks compilation and interpretation traces, then ships it as a locally-runnable Ollama model so users can run Playbooks programs without paying frontier-model token costs. Pipeline: extract traces from LangFuse → curate prompt/response pairs with stratified splits → LoRA-fine-tune on AWS SageMaker (ml.g5.xlarge, single GPU) → convert LoRA adapter to GGUF → serve via Ollama. Both Qwen-3-4B and Gemma training paths exist. Status: experimental, paused alongside the main Playbooks project.

## Why notable

- Full vertical: production telemetry → curated dataset → cloud fine-tuning → quantized local serving.
- Practical economics — fine-tunes the interpreter (the hot path) while keeping a frontier model for compilation (the cold path).
- Mixed-cloud / mixed-format work: LangFuse, SageMaker, Docker, GGUF, Ollama.

## Architecture

- `src/data/` — `extract_full_dataset.py` pulls Playbooks-related traces/generations from LangFuse; `create_training_dataset.py` does dedup, quality scoring, ChatML formatting, and stratified train/val/test splits.
- `src/training/` — SageMaker entry points (`train.py`, `train_refactored.py`, `train-gemma.py`, `train-gemma-lora.py`), `Dockerfile`, `launch_sagemaker_training.py`, `setup_sagemaker.sh`, `setup_macbook.sh`, `build_and_push.sh`, plus `local_convert_to_gguf.py` for adapter export.
- `src/local_model_inference/` — `Modelfile.base` and `Modelfile.playbookslm` for Ollama.
- `src/ec2/` — EC2 debug-instance helpers (`launch_ec2_debug.py`, `download_training_files.sh`).
- Configured to point Playbooks at the local model via `MODEL=ollama/playbooks-lm` + `LLM_API_BASE=http://localhost:11434` (keeping Sonnet as the compiler).
