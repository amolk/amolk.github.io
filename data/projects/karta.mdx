Karta is a thin orchestration layer for production AI agents. It builds on top of existing coding agents - Claude Code, OpenCode - rather than reimplementing them, and adds the pieces real applications need: multi-user sessions, multi-agent routing, participant-aware messaging, cross-instance communication, HTTP deployment, multi-tenancy, policy enforcement, and lifecycle hooks. The bet is that the next generation of agentic apps will be assembled from harnesses that already exist, and what's missing is the surrounding control plane.

The runtime ships on PyPI as `karta-python` and pairs with `karta-web`, a Rails control plane handling identity, organizations, audit logs, and per-org API keys. The two halves are wired through a service-token channel so customer keys are validated centrally while agent execution stays in the runtime. It is the active flagship of the work - a monorepo with both halves under one roof.

Agent execution can run locally or on a managed cloud runtime, with each session isolated in its own microVM on AWS Bedrock AgentCore. Both paths sit behind a single execution-backend seam, so local development stays identical to production while the cloud path adds durable per-session workspaces, per-user credential isolation, and hosted MCP gateways. The full loop runs end to end: deploy a harness app, then chat with it from a product's own UI over streaming HTTP, with replies coming straight from the agent.
