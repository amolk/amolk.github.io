Karta is a managed agent harness platform - Heroku, but for harness applications. An agent harness is the agent runtime itself - Claude Code or OpenCode - and a harness application is just a folder that configures one, a `.claude/` directory and a `CLAUDE.md`. You bring the harness application; Karta deploys and runs it for you - securely, scalably, and reliably, with per-user isolation and built-in usage and cost controls. The bet is that the next generation of agentic apps will be assembled from harnesses that already exist, and what's missing is the platform around them.

Rather than reimplementing agent capabilities, Karta adds the pieces real products need around them. Every Karta project exposes the same session protocol - create a session, send a message, stream the response - so any front end talks to any agent the same way. The platform provides multi-user sessions, multi-tenancy, identity, billing, usage metering, budget enforcement, BYOK, and webhooks out of the box. Agent code runs isolated and per-user, and by design the part of the system that runs your agent holds none of your money-and-identity state. Each user's workspace is durable too, persisting across sessions and across new deploys of the app, while compute stays ephemeral and per-session.

Publishing a harness app creates a versioned release at a stable endpoint, `agent.karta.sh/<org>/<project>`. Getting an agent live takes one of three on-ramps. `git push` deploys a folder straight from your machine, with build progress streaming back over the push itself and ending in the live agent URL. A browser flow creates and edits an agent - including its `CLAUDE.md` - with no terminal at all, where every save becomes a new versioned release. Or a single `karta` CLI carries you from `init` through a local dev loop to a live URL. And any product can embed a Karta-backed chat widget with a publishable key, in plain JavaScript or as a React component, with the agent's replies streaming straight into the page.
