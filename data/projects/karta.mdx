## About

Karta is a thin orchestration layer that builds on top of existing coding agents (Claude Code, OpenCode) rather than reinventing them. It adds what production applications need: multi-user sessions, multi-agent routing, participant-aware messaging, cross-instance communication, HTTP deployment, multi-tenancy, policy enforcement, and lifecycle hooks — without reimplementing agent capabilities. Install: `pip install karta-python`.

## Architecture

```
apps/
  karta-python/    Python runtime: sessions, agents, policies, hooks, workspace strategies, MCP
  karta-web/       Rails control plane: identity, organizations, audit log, dashboard, per-org API keys
```

Harness Layer (Claude Code / OpenCode) → Karta Core (sessions, agents, policies, hooks) → Client Layer (HTTP API, CLI, Python SDK) → Gateway (event ingestion, fan-out) → Multi-Tenancy (KartaHub, WorkspaceManager). karta-python validates customer API keys against karta-web via `/v1/internal/tenants` over a service-token channel.
