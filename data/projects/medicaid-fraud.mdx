## About

Investigative project that maps publicly available CMS and federal healthcare datasets to concrete Medicaid fraud detection hypotheses. Center of the work is a comprehensive supplementary dataset catalog covering 29 dataset categories and ~50M+ rows (~25GB), with each dataset annotated by its fraud-investigation applications (identity verification, ownership-network analysis, dual-billing detection, prescribing-pattern analysis, etc.) and tied back to 12+ named investigations (FL spike, inactive providers, shell companies, controlled-substance prescribing, …). The data corpus itself lives outside the tracked tree; the catalog is the durable artifact.

## Why notable

- Domain-deep work in a regulated, high-stakes area; reflects serious engagement with CMS data shapes, not a toy demo.
- Catalog-first methodology — knowing what data exists, where it lives, and which hypothesis it answers — is exactly the bottleneck in real-world fraud work.
- Cross-dataset linkage thinking (EIN clusters, authorized-official networks, NPI deactivation timelines) shows graph-style investigative instincts.

## Architecture

- `investigations/dataset-catalog.md` (~1170 lines) — tiered (Tier 1/2/3) inventory: NPPES (9.4M providers, 330 columns), SAM Exclusions, Open Payments, Medicare Enrollment, Part D Prescribers, HHA Ownership, plus 20+ more.
- Phase plan: Week 1 (Tier-1 foundation load) → Week 2 (Tier-2 analysis) → Week 3+ (Tier-3 on-demand).
- Each dataset entry documents: file path, size, row/column counts, key columns, fraud applications, and which named investigation it strengthens.
- No code yet; the catalog is upstream of any pipeline build-out.
