## About

Deterministic Python implementation of the **Decision Intelligence framework** specified in carapace-intelligence's `docs/decision-intelligence-paper.md` (2026-05-23). The LLM is treated as a *learned environment model* — a stochastic transition operator + reward elicitor — and explicit search, Bayesian expectimax backward induction, and structured robustness analysis are layered on top. All global reasoning (value, recommendation, sensitivity) is deterministic and reproducible from cached LLM outputs.

**Pivot history.** Started 2026-04-23 as a workspace for a "query-first, markdown-native, STAG-shaped" rethink of carapace-intelligence (`PRD.md` + `IMPLEMENTATION.md`, no git). Sat dormant through May until 2026-05-27, when the workspace was re-scoped and bootstrapped as a real repo to implement the DI paper as code in a single overnight run. The earlier STAG framing did not carry into the new codebase.

## Architecture

```
carapace-intelligence2/
  decision-intelligence-paper.md   spec (mirrors carapace-intelligence/docs/)
  src/di/
    core/         types, hashing, graph, config, taxonomy
    cache/        content-addressed store with mandatory rationale sidecars
    solver/       value, recommend, subopt, best-line, opponent modes
    sensitivity/  four-arm perturbation sweep + pivot conditions
    agents/       LLM elicitation agents (framer, typer, tempo-holder, …)
    construct/    full-BFS + PUCT-MCTS tree builders
    audit/        tail-risk gate
    prescribe/    final narrative + evidence map
    orchestrator.py
    cli.py
  project/
    action_kinds.yaml            project-pinned action taxonomy
    tail_risk_patterns.yaml      project-pinned pattern library
  tests/
    fixtures/    hand-built graphs for solver/sensitivity validation
    unit/        per-module
```
