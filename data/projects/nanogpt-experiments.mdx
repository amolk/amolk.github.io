## About

A fork of Karpathy's nanoGPT used as a sandbox for an "adaptive compute" transformer architecture. The core idea: train each block individually (in addition to in-sequence), have consecutive blocks vote via an LIF-style evidence accumulator on whether to emit a token, and let the stack recur until a stable winner emerges. The README captures extensive design notes on splitting blocks into sensory / internal (recurrent) / motor stages, noise injection for Monte-Carlo-like search, embedding decomposition (sensory / internal / motor / goal representation), and how all of this ties into Amol's cognitive-homeostasis / hvar–cvar framing from i3AI.

## Why notable

- Original adaptive-compute design that predates and parallels the broader industry interest in test-time-compute / chain-of-thought-style recurrence.
- Directly connects the i3AI cognitive-homeostasis research (cvars, hvars, reward prediction) to a concrete transformer architecture proposal.
- The `abab` / `patternloop` custom datasets show willingness to build minimal probes for the specific phenomena under study.

## Architecture

- **Base codebase**: standard nanoGPT layout — `model.py` (GPT block), `train.py`, `sample.py`, `bench.py`, `configurator.py`.
- **Configs** (`config/`): `train_gpt2.py`, `train_shakespeare_char.py`, `train_ruby_char.py`, `finetune_shakespeare.py`, plus `eval_gpt2*.py` for the four GPT-2 sizes.
- **Data preps** (`data/`): standard `shakespeare`, `shakespeare_char`, `openwebtext`, plus two custom toy datasets — `abab` and `patternloop` — built for adaptive-compute / pattern-loop probing.
- **Notebooks**: `scaling_laws.ipynb`, `transformer_sizing.ipynb` for back-of-envelope analysis.
- **Architectural experiments**: live primarily in the README design notes (per-block LIF, recurrent loops 3rd→Nth block, noise-as-MC-search, weighted-by-predicted-reward LIF) rather than as committed model.py forks — this is a thinking sandbox, not a polished system.
- **Run**: `python data/<dataset>/prepare.py` then `python train.py config/<config>.py`.
