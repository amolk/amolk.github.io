## About

agent-callstack gives AI agents a proper call stack. Each `/call` forks the parent's full session context — like `fork()` in Unix — so the child inherits everything without re-prompting. When the child completes, only its compact return value flows back; the parent's context stays clean. The design solves two failure modes: sub-agents that start blind (no parent context, wasted tokens) and main-agent accumulation (context rot, lossy compaction). The core insight was first built in Playbooks AI (2023) and is here generalized to work with any harness, starting with Claude Code.

## Architecture

```
plugins/callstack/agent_callstack/
  __init__.py    Public API: call, call_many, resume, Caller, Result
  state.py       Pure state machine: discriminated unions + step()
  driver.py      Effect runner: ties channel + state + tree
  channel.py     Claude CLI subprocess + NDJSON protocol
  protocol.py    SYSTEM_INSTRUCTION + envelope parser
  session.py     SessionLocator: ~/.claude/projects discovery
  trace.py       TraceWriter (JSONL) + TreeStore (sidecar snapshots)
  analysis.py    SessionAnalyzer: post-execution inspection
.claude/skills/call/   Claude Code skill entrypoint
examples/
  customer_support/  MFA + refund workflow demo
  parallel_calls/    Fan-out with nested parallelism demo
```

Install: `cp -r agent-callstack/.claude/skills/call ~/.claude/skills/`
