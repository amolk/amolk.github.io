## About

AgentZero is a meta-agent: it bootstraps new multi-agent projects (agents, sub-agents, skills) and then reviews, simulates, and scores them. It is intentionally architecture-aware — it makes the three-level decomposition decisions itself rather than asking the user. Python 3.12 (Poetry-managed), targets multiple agent harnesses (Claude Code and OpenCode). Built in a tight Feb–Mar 2026 sprint as an exploration of self-scaffolding agent infrastructure.

## Why notable

- Self-referential: an agent whose job is to design other agents, with explicit architecture-decision authority.
- Multi-harness from day one — portable across Claude Code and OpenCode rather than locked to one vendor.
- Couples scaffolding with simulation + scoring, so generated projects get an automated quality signal.

## Architecture

- `src/agent_zero/` — Python package (Poetry, `pyproject.toml`, `agent-zero` 0.1.0, requires Python ≥3.12).
- `dev/` — internal tooling, skills, and templates that AgentZero uses while scaffolding.
- Workflow primitives observable in commit history: `/new-project`, `/add-agent`, simulation skill with transcript analysis, post-scaffold review loop, two-tier skill-authoring templates, tenant worktrees for multi-org projects.
- Multi-harness adapter layer lets the same project artifacts run under Claude Code or OpenCode.
- Run via Poetry (`poetry install`, then invoke the harness with AgentZero's skills/commands loaded).
